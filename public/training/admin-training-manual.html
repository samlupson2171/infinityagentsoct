1. Introduction

Welcome to the Infinity Weekends Admin Dashboard — the central management hub for the entire Infinity Weekends platform. As an administrator, you have full control over enquiries, quotes, bookings, destinations, packages, events, users, and system settings.

Admin Capabilities Overview

  • View and manage all agent enquiries across the platform
  • Create, edit, and send professional quotes to agents
  • Track bookings and manage their lifecycle
  • Manage destinations, offers, and Super Packages
  • Create and manage events and activities
  • Approve or reject agent registrations
  • Manage users, roles, and permissions
  • Upload and organise training materials for agents
  • Configure email settings and system preferences
  • Monitor platform analytics and performance metrics

Admin Roles

RoleAccess LevelDescription

AdminOperational AccessManage enquiries, quotes, bookings, content, and approve agents
Note: This manual covers all admin features. Some features may be restricted based on your specific role and permissions.

Pricing Matrix Configuration

The pricing matrix is the heart of a Super Package. It defines per-person prices based on three dimensions:

1. Travel Period

Define pricing for different months or seasons:

  • Individual months (January, February, etc.)
  • Seasonal groupings (e.g., "Peak Summer: June-August")
  • Custom date ranges

2. Group Size Tiers

Set different prices for different group sizes:

TierExample RangeRationale
Small8-12 guestsHigher per-person cost due to fewer people sharing fixed costs
Medium13-20 guestsMid-range pricing with moderate group discounts
Large21+ guestsBest per-person rate due to economies of scale

3. Duration

Prices for different trip lengths:

  • 2 nights
  • 3 nights
  • 4 nights

Setting Up the Matrix

1 In the package editor, navigate to the Pricing tab
2 Define your travel periods (add months or date ranges)
3 Define your group size tiers (set min/max for each tier)
4 For each combination of period × group size × duration, enter the per-person price
5 The matrix auto-generates a grid for easy data entry
Tip: Start with 3 group size tiers and 3 durations. This gives you a manageable 9-cell matrix per period. You can always add more granularity later.

16. Troubleshooting & FAQs

Emails are not being sent

  • Check email settings in /admin/settings
  • Verify the API key is correct and active
  • Check the email service dashboard (e.g., Resend) for delivery errors
  • Ensure the "From" email domain is verified with your email provider
  • Send a test email to diagnose the issue
  • Check server logs for error messages

Agent cannot log in after approval

  • Verify their account status is "Approved" or "Contracted" in User Management
  • Check if they need to sign a contract first (if contract requirement is enabled)
  • Ensure they're using the correct email address
  • Try resetting their password from the admin panel
  • Check if their account was accidentally set to "Rejected"

Quote prices showing incorrectly

  • Verify the pricing in the quote editor — check total and per-person calculations
  • Ensure the number of guests is correct (price per person = total ÷ guests)
  • If linked to a Super Package, check the package's pricing matrix
  • Clear browser cache and refresh the page

Super Package prices showing "Not Available"

  • Open the package and check the pricing matrix
  • Ensure all cells in the matrix have values (empty cells show "Not Available")
  • Verify the travel period, group size tier, and duration combination exists
  • Check that the package status is "Active"

CSV import failing

  • Download and use the provided CSV template
  • Ensure all required columns are present
  • Check for special characters that may break parsing (commas in fields should be quoted)
  • Verify date formats match the expected format (YYYY-MM-DD)
  • Check file encoding is UTF-8
  • Ensure the file size doesn't exceed the upload limit

Dashboard metrics not updating

  • Refresh the page (Ctrl+R or Cmd+R)
  • Clear browser cache
  • Check if there are any server errors in the browser console
  • Metrics update in real-time — if they're stale, there may be a database connection issue

Agent registration emails not arriving

  • Check the agent's spam/junk folder
  • Verify the email address is correct in their registration
  • Check email settings and service status
  • Manually resend the notification from the approvals page

Cannot delete a destination/package

  • Check if there are active quotes or enquiries referencing the item
  • Use soft delete (archive) instead of hard delete for items with references
  • Deactivate the item to hide it from agents without deleting

Frequently Asked Questions

Q: Can I undo a rejected agent approval?

A: Yes. Go to User Management, find the agent, and change their status from "Rejected" to "Approved". They will need to be notified manually that their account is now active.

Q: How do I change the email template design?

A: Email templates are managed in the codebase. Contact your development team to make changes to email templates. Test thoroughly after any changes.

Q: Can multiple admins work on the same enquiry?

A: Yes, but be aware that simultaneous edits may overwrite each other. Communicate with your team about who is handling which enquiries.

Q: What happens when a quote expires?

A: If auto-expire is enabled, the quote status changes to "Expired" and the agent can no longer accept it. You can create a new quote or extend the validity by editing and resending.

Q: How do I add a new admin user?

A: Go to User Management → Add User → Set role to "Admin" or "Super Admin". The new admin can log in immediately with the credentials you set.

Q: Can agents see draft quotes?

A: No. Quotes in "Draft" status are only visible to admins. Agents can only see quotes that have been sent (status: "Sent" or "Updated").

Q: How do I handle a booking cancellation?

A: Open the booking, change the status to "Cancelled". The agent will be notified. Document the reason in the booking notes for your records.

Q: Is there a limit to how many Super Packages I can create?

A: No. You can create as many packages as needed. However, keep the catalogue manageable for agents — consider archiving outdated packages rather than keeping them all active.
